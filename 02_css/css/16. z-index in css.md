The CSS `z-index` property controls the vertical stacking order of elements that overlap. If you think of your screen as having an X-axis (left to right) and a Y-axis (top to bottom), the `z-index` controls the **Z-axis** (front to back, pointing directly at your eyes).

When elements overlap, the browser needs to know which one should appear on top of the other. `z-index` is how you make that decision.

---

### **The Golden Rule of `z-index`**

`z-index` **only works on positioned elements**.

If an element has the default `position: static`, any `z-index` you give it will be completely ignored. You must set the element's `position` to `relative`, `absolute`, `fixed`, or `sticky` for `z-index` to take effect.

### **How It Works**

- **Higher numbers go on top:** An element with `z-index: 2` will cover an element with `z-index: 1`.
    
- **Default value:** By default, elements are layered based on their order in the HTML code (elements further down in the code appear on top). The default `z-index` value is `auto` (which effectively acts like `0`).
    
- **Negative numbers:** You can use negative numbers (like `z-index: -1`) to push an element behind others, often used to place a background element behind standard text.
    

---

### **Putting It Together (Runnable Example)**

In this example, we have three overlapping boxes. Normally, Box 3 (being last in the HTML) would be on the very top. However, we are going to use `z-index` to force Box 2 to be the top layer.

HTML

```
<!DOCTYPE html>
<html lang="en">
<head>
    <style>
        body {
            font-family: sans-serif;
            background-color: #f4f4f9;
            padding: 40px;
        }

        .container {
            position: relative; /* Creates the boundary */
            height: 300px;
        }

        /* Shared styles for all boxes */
        .box {
            position: absolute; /* Allows them to overlap */
            width: 150px;
            height: 150px;
            color: white;
            font-weight: bold;
            display: flex;
            align-items: center;
            justify-content: center;
            border-radius: 8px;
            box-shadow: 0 4px 10px rgba(0,0,0,0.3);
            border: 2px solid white;
        }

        /* Box 1: The Bottom Layer */
        .box1 {
            background-color: #ff7e5f;
            top: 20px;
            left: 20px;
            z-index: 1; 
        }

        /* Box 2: The Top Layer (Highest z-index) */
        .box2 {
            background-color: #3a7bd5;
            top: 70px;
            left: 70px;
            z-index: 99; /* This huge number guarantees it sits on top */
        }

        /* Box 3: The Middle Layer */
        .box3 {
            background-color: #4ca1af;
            top: 120px;
            left: 120px;
            z-index: 2; /* Higher than Box 1, but lower than Box 2 */
        }
    </style>
</head>
<body>

    <div class="container">
        <div class="box box1">Box 1<br>(z-index: 1)</div>
        <div class="box box2">Box 2<br>(z-index: 99)</div>
        <div class="box box3">Box 3<br>(z-index: 2)</div>
    </div>

</body>
</html>
```

### **A Quick Note on "Stacking Contexts"**

Sometimes, a `z-index` of `9999` still won't put an element on top. This happens because certain properties (like `opacity`, `transform`, or being inside a parent with its own `z-index`) create a "stacking context." Think of it like a folder—if Box A is in a folder that is physically behind Box B's folder, Box A can never overlap Box B, no matter how high Box A's `z-index` is.
