n CSS, the `overflow` property acts as the "traffic controller" for your layout. It tells the browser what to do when an element's content is too large to fit inside its specified box (the container).

Think of it like trying to fit a king-sized mattress into a studio apartment—you either have to trim the mattress (`hidden`) or add a hallway so people can move around it (`scroll`).

---

## 1. Overflow: Hidden

When you set `overflow: hidden`, any content that extends beyond the element's width or height is **clipped**. The extra content is still there in the code, but it is completely invisible to the user, and they cannot scroll to see it.

- **Best for:** Cleaning up messy layouts, preventing "leakage" of images, or creating specific UI effects like rounded corners where the content inside needs to stay "inside the lines."
    

## 2. Overflow: Scroll

The `scroll` value forces scrollbars to appear on the element. This allows the user to see the "hidden" content by moving the bar.

- **Note:** A small quirk of `scroll` is that it usually shows both horizontal and vertical scrollbars, even if the content only overflows in one direction. Most developers actually prefer `overflow: auto`, which only shows scrollbars **if** the content is actually too big.
