The `cursor` property is a simple but incredibly important tool for User Experience (UX). It tells the browser what the mouse pointer should look like when a user hovers over an element, giving them a visual clue about what they can do (like clicking, typing, or dragging).

### The Syntax

CSS

```
.my-element {
  cursor: pointer; 
}
```

---

### Common Cursor Values (Categorized for Notes)

There are dozens of cursor types, but you will use these core ones 95% of the time:

#### 1. The Essentials

- **`cursor: default;`** The standard arrow pointer. Use this if you want to force the normal arrow.
    
- **`cursor: pointer;`** The "pointing hand." **Crucial Note:** You should always add this to buttons, custom links, and anything the user can click.
    
- **`cursor: none;`** Completely hides the mouse pointer when it's over the element.
    

#### 2. Status & Information

- **`cursor: wait;`** The loading hourglass or spinning wheel. Tells the user the background is processing something and they should wait.
    
- **`cursor: progress;`** Similar to wait, but usually an arrow _with_ a spinning wheel, meaning "loading, but you can still click other things."
    
- **`cursor: help;`** An arrow with a question mark. Great for tooltips or informational icons.
    
- **`cursor: not-allowed;`** The "red circle with a slash." Tells the user a button is disabled or an action cannot be performed.
    

#### 3. Text & Interaction

- **`cursor: text;`** The "I-beam" cursor. Shows the user they can highlight or type text here.
    
- **`cursor: crosshair;`** A plus sign. Often used for drawing apps or selecting a specific coordinate on an image.
    

#### 4. Movement & Dragging

- **`cursor: move;`** Four arrows pointing in all directions. Indicates the element can be moved around.
    
- **`cursor: grab;`** An open hand. Indicates an element can be grabbed and dragged (like a map).
    
- **`cursor: grabbing;`** A closed hand. Use this in combination with `:active` to show the user is actively holding/dragging the item.
