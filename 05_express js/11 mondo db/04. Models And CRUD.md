hink of a **Model** as the bridge between your Express code and your MongoDB database. While the **Schema** is the "blueprint" (defining what a user looks like), the **Model** is the actual "worker" that has methods to Create, Read, Update, and Delete data.

---

### 1. The Model (`models/ClientModel.js`)

This defines the structure and creates the "Worker" (Model).



```JavaScript
import mongoose from "mongoose";

const clientSchema = new mongoose.Schema({
    name: { type: String, required: true },
    email: { type: String, required: true, unique: true },
    project: String,
    status: { type: String, default: "Pending" }
}, { timestamps: true });

const Client = mongoose.model("Client", clientSchema);
export default Client;
```

---

### 2. The Controller (`controllers/clientController.js`)

This is where the **CRUD** logic lives using Model methods.



```JavaScript
import Client from "../models/ClientModel.js";

// 1. CREATE
export const createClient = async (req, res) => {
    try {
        const client = await Client.create(req.body);
        res.status(201).json(client);
    } catch (err) {
        res.status(400).json({ error: err.message });
    }
};

// 2. READ (Get All)
export const getAllClients = async (req, res) => {
    try {
        const clients = await Client.find();
        res.status(200).json(clients);
    } catch (err) {
        res.status(500).json({ error: err.message });
    }
};

// 3. UPDATE (By ID)
export const updateClient = async (req, res) => {
    try {
        const updatedClient = await Client.findByIdAndUpdate(
            req.params.id, 
            req.body, 
            { returnDocument: 'after' } // Returns the updated document instead of the old one
        );
        res.status(200).json(updatedClient);
    } catch (err) {
        res.status(400).json({ error: err.message });
    }
};

// 4. DELETE
export const deleteClient = async (req, res) => {
    try {
        await Client.findByIdAndDelete(req.params.id);
        res.status(200).json({ message: "Client Deleted" });
    } catch (err) {
        res.status(500).json({ error: err.message });
    }
};
```

---

### 3. The Routes (`routes/clientRoutes.js`)

Connect your URLs to the controller functions.

JavaScript

```JavaScript
import express from "express";
import { 
    createClient, 
    getAllClients, 
    updateClient, 
    deleteClient 
} from "../controllers/clientController.js";

const router = express.Router();

router.post("/", createClient);        // POST /api/clients
router.get("/", getAllClients);        // GET /api/clients
router.put("/:id", updateClient);      // PUT /api/clients/ID
router.delete("/:id", deleteClient);   // DELETE /api/clients/ID

export default router;
```

---

### 4. Connect to `app.js`

Don't forget to register the routes in your main `app.js`:

JavaScript

```JavaScript
import clientRoutes from "./routes/clientRoutes.js";

// ... existing code
app.use("/api/clients", clientRoutes);
```

---

### Summary of Model Methods used:

|Operation|Model Method|Description|
|---|---|---|
|**Create**|`Model.create()`|Saves a new document to the collection.|
|**Read**|`Model.find()`|Retrieves documents (all or filtered).|
|**Update**|`Model.findByIdAndUpdate()`|Finds a specific doc by ID and changes it.|
|**Delete**|`Model.findByIdAndDelete()`|Removes a specific doc from the database.|
